Git对象

Git的核心部分是一个简单的键值对数据库，可以向git仓库中欧给你插入任意类型的数据，他会返回一个唯一的键，通过键可以在任意时刻再次取回该内容。

可以通过底层命令 git has-object 来实现效果，该命令可以将任意数据保存于 .git/objects 目录下，并返回该对象的键值。

键值是通过你在 git init 时给定的 --object-format 参数来生成的，默认是sha1，也就是说所谓的键值实际上是数据的hash值。

是否有可能产生意料之外的hash碰撞，答案是会的，但是第一这种行为除非刻意去出发，正常情况下很难发生，其次git并只将数据做hash，同时也会用一个头部信息和数据合并，同时计算出一个hash值，使碰撞发生的可能性大大降低。

注：hash生成方式： sha1([type] [length]\0[content])
生成出的校验值，git将会使用前两位命名子目录，后38位用作文件名。

可以通过 git cat-file <key> 命令来取回存储的数据。
-p 动态获取 -t 获取类型

树对象（tree object）:

它能解决文件名保存的问题，也允许我们将多个文件组织到一起。 Git 以一种类似于 UNIX 文件系统的方式存储内容，但作了些许简化。 所有内容均以树对象和数据对象的形式存储，其中树对象对应了 UNIX 中的目录项，数据对象则大致上对应了 inodes 或文件内容。 一个树对象包含了一条或多条树对象记录（tree entry），每条记录含有一个指向数据对象或者子树对象的 SHA-1 指针，以及相应的模式、类型、文件名信息。

Git 内部原理10.1 笔记

看来直接硬啃文档不可取，学习命令的时候对整个git的内部原理会有一些疑惑，比如在git clone –shared中，实操发现区别在于 .git/objects 文件夹下，但是这个文件夹时做什么用的呢？存的是改动吗？使用了shared参数以后就会直接使用/info/alternates文件下给定的路径吗？

在执行 git init 建立非裸仓库时，会建立一个 .git 目录，这个目录里包含了所有git存储和操作的东西，如果像备份或复制一个版本库，可以把整个目录拷贝至另一处即可。
目录的典型结构为，不同的版本可能包含其他内容：

其中：

  • description文件用于共GitWeb程序使用；
  • config文件包含项目特有的配置选项；
  • info目录包含一个全局性排除文件，用于防止那些不希望被记录在 .gitignore 文件中的忽略模式；
  • hooks目录包含客户端或服务端的钩子脚本；

重要部分：

  • HEAD文件指向目前被checkout的分支
  • refs目录存储指向数据的提交对象的指针；
  • index文件保存暂存区信息；
  • objects目录存储所有的数据内容。

Git学习笔记-DAY2

克隆 git clone \<repository\> [\<directory\>]

克隆一个仓库到新建的文件夹中，对于克隆的库里的每一个分支，建立一个远程追踪分支（可以被git branch –remotes看见的），然后建立一个初始分支来自于被克隆库当前的分支，并切换到它上面。

在克隆后一个 git fetch 可以更新所有远程追踪分支，一个 git pull 会在此基础之上合并远程和本地的主分支。

通过获取远程分支的head引用（在 refs/remotes/origin 中），并通过初始化 remote.origin.url 和 remote.origin.fetch 来建立默认配置的。

参数

-l –local

本地克隆，当你从本地克隆一个库时，这个参数将会告诉git复制头指针和所有在objects，refs文件夹下的所有东西，而在.git/objects文件夹下的文件将会是硬链接的一节省磁盘空间。
如果库被指向了一个本地路径，那这个参数将被默认启用，加入这个参数等于无用功，如果库指向了一个URL，那么这个参数将被默认忽略，如果制定了 --no-local 参数将会覆盖默认值。

-no-hardlinks

强制克隆本地库的操作复制在 .git/objects 文件夹下的文件而不是用硬链接，当你希望给本地库做备份的时候可以使用这个命令。

-s –shared

当本地机器上的一个库被克隆时，设置 .git/objects/info/alternates 与源库共享objects来取代硬链接（符号连接？），最后克隆出来的库将会没有任何自己拥有的对象。

注：这可能是个很危险的操作，不要使用它除非你很清楚将会发生什么。如果你克隆你的库使用这个选项然后删除了源库中的分支，或使用其他git命令是的任何提交变成未引用的或者悬空的，这些对象可能被正常的Git操作移除。

Git学习笔记 DAY1

前言

准备学习一下Git的命令行，虽说现在用的是perforce，但是说不准那天就要用上。
正好可以利用这个机会顺手玩一下Hexo，这个站就是用Hexo搭的，配合GitHub Page，正好可以学习Git的使用。
笔记大部分情况是给自己看的，所以我也就不讲什么是git了，没什么太大的意义。
我的学习路线是直接看git自己的文档顺便做一个翻译，手敲命令去理解，然后对于不懂的部分再去google查对应的应用，然后作为笔记记在对应的parameter上，第一天，先来看一个init吧。

创建本地仓库 git init

这个命令主要用来创建一个新的Git Repository，本质上使在文件夹中建立一个 .git 文件夹，这个文件夹将会作为你的本地仓库使用

参数：

-q –quite：

仅打印错误和警告消息；所有其他输出不会显示

–bare：

在当前目录内创建一个裸库，不建立工作区。

裸库被拉取后可以被正常的被clone和push更新，但是因为没有工作区在没有clone前不能进行commit，而非裸库在clone后则无法直接提交修改给非裸库，会得到如下的错误：

error: refusing to update checked out branch: refs/heads/master

error: By default, updating the current branch in a non-bare repository
is denied, because it will make the index and work tree inconsistent
with what you pushed, and will require ‘git reset –hard’ to match
the work tree to HEAD.

故一般用于建立远程库。

–object-format=<sha1/sha256>：

指定存储库的hash算法，默认值为 sha1 ；

–template=<directory>：

使用模板创建工作目录，默认值为按如下优先级顺序获取:

  • 指定参数<directory>;
  • GIT_TEMPLATE_DIR 环境变量；
  • init.templateDir 配置变量；
  • 默认模板

创建库时会将其中的所有不以. 为开头的文件和目录复制进库中，默认的模板中有一些hook的样例，但是这些hook不会被启用，需要启用则需要删除其后缀 .sample 。
另外还有一些建议的排除模式在 info/exclude 下。

–separate-git-dir=<directory>

将本地库初始化至指定目录，也可以用于本地库移动到指定的路径。

这个和bare repository的区别，同样或创作工作区，适用于将本地库和工作区分开。

-b –initial-branch=<name>

指定新建库中的初始分支名字，默认值为 master 。

–shared[=(false|true|umask|group|all|world|everybody|0xxx)]

指定git库的用户权限，默认值为umask或false

  • umask: 使用当前目录的权限
  • group：使用当前目录的权限，并设置组间可读写
  • all: 使用当前目录的权限，并设置所有用户可读写
  • 0xxx: 指定权限。
