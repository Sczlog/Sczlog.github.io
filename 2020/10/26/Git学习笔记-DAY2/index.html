Git学习笔记-DAY2

克隆 git clone \<repository\> [\<directory\>]

克隆一个仓库到新建的文件夹中，对于克隆的库里的每一个分支，建立一个远程追踪分支（可以被git branch –remotes看见的），然后建立一个初始分支来自于被克隆库当前的分支，并切换到它上面。

在克隆后一个 git fetch 可以更新所有远程追踪分支，一个 git pull 会在此基础之上合并远程和本地的主分支。

通过获取远程分支的head引用（在 refs/remotes/origin 中），并通过初始化 remote.origin.url 和 remote.origin.fetch 来建立默认配置的。

参数

-l –local

本地克隆，当你从本地克隆一个库时，这个参数将会告诉git复制头指针和所有在objects，refs文件夹下的所有东西，而在.git/objects文件夹下的文件将会是硬链接的一节省磁盘空间。
如果库被指向了一个本地路径，那这个参数将被默认启用，加入这个参数等于无用功，如果库指向了一个URL，那么这个参数将被默认忽略，如果制定了 --no-local 参数将会覆盖默认值。

-no-hardlinks

强制克隆本地库的操作复制在 .git/objects 文件夹下的文件而不是用硬链接，当你希望给本地库做备份的时候可以使用这个命令。

-s –shared

当本地机器上的一个库被克隆时，设置 .git/objects/info/alternates 与源库共享objects来取代硬链接（符号连接？），最后克隆出来的库将会没有任何自己拥有的对象。

注：这可能是个很危险的操作，不要使用它除非你很清楚将会发生什么。如果你克隆你的库使用这个选项然后删除了源库中的分支，或使用其他git命令是的任何提交变成未引用的或者悬空的，这些对象可能被正常的Git操作移除。
